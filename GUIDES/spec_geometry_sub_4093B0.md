# Спека: `sub_4093B0` — генерация вершин в динамический буфер (`dword_4D7E70`)

Цель: зафиксировать “истину” о том, что на самом деле делает `sub_4093B0` в оригинале (формат вершин, UV/normal, условия вызова, связь с рендером).

---

## 0) Ключевой вывод

`sub_4093B0` **не** “строит меш сегмента ветки”. Это **заполняет (lock/write/unlock) динамический GPU-буфер** `dword_4D7E70` вершинами фиксированного формата, используя `flt_4D7E88` как коэффициент масштаба/интенсивности, и сбрасывает его в 0 после записи.

---

## 1) Условия входа и глобалы

Файл: `decompiled/sub_4093B0.c`

Вход:
- без параметров; использует глобалы:
  - `flt_4D7E88` (должно быть `!= 0`)
  - `dword_4D7E70` (объект буфера, имеет методы `+44` lock и `+48` unlock)
  - `dword_4D7E78` (счётчик записанных вершин в текущей “заливке”)

Условие выполнения:
- `if (flt_4D7E88 != 0 && lock(dword_4D7E70, &v43) >= 0) { ... }`

После завершения:
- `unlock(dword_4D7E70)`
- `dword_4D7E78 = max(0, dword_4D7E78 - 2)` (в коде: если <=2, то 0, иначе -2)
- `flt_4D7E88 = 0.0`

---

## 2) Формат вершины (stride)

Запись идёт по `float* v43`, с инкрементом `v43 += 8` на вершину.

Формат (8 float):
- `v43[0..2]` — position.xyz
- `v43[3..5]` — normal.xyz
- `v43[6..7]` — uv.xy

Stride:
- `8 * 4 = 32 bytes` (подтверждается тем, что при submit через `sub_44F2A0` используется `dword_4D7E80 = 32` как `a7`).

FVF:
- `dword_4D7E84 = 274 (0x112) = D3DFVF_XYZ (0x2) | D3DFVF_NORMAL (0x10) | D3DFVF_TEX1 (0x100)`.

Primitive:
- `dword_4D7E74 = 5` (используется как `a6` в `sub_44F2A0`) → `D3DPT_TRIANGLESTRIP`.
- `dword_4D7E78` передаётся как `a5` и в `sub_4093B0` после записи делается `count -= 2`, т.е. это **primitiveCount** для triangle strip (vertexCount - 2).

UV формула (концентрическая проекция):
- угол: \(\theta = t * 2\pi\) (в коде `v45 * 6.283185...`)
- `u = sin(theta) * scale + 0.5`
- `v = cos(theta) * scale + 0.5`
- `scale` меняется по сегменту: базово `v50` и “внутренний” `v47` (оба вычисляются из `flt_4D7E88 * 0.03` и прогресса вдоль “ленты”).

---

## 3) Геометрическая схема

Внутренние циклы:
- внешний “кольцевой” индекс `v46` от 0 до 1 (две части)
- внутренний `v9` от 0 до 32, шаг `0.03125` (то есть 33 шага)
- на каждом шаге пишутся **две вершины** (двойная запись: сначала одна, затем вторая с другим scale `v47`).

Для расчёта базисов/нормалей используются:
- `D3DXMatrixRotationYawPitchRoll` (поворот вокруг yaw)
- `sub_4085B0` и `sub_401540` (координатные преобразования / нормали)
  - `sub_4085B0(a1, out3, m4x4)` = `D3DXVec3TransformCoord(out3, a1, m4x4)`
  - `sub_401540(a1, out3, m4x4)` = `D3DXVec3TransformNormal(out3, a1, m4x4)`

## 4) Где и как вызывается

### 4.1 Инициализация буфера и первичное заполнение — `sub_409910`

Файл: `decompiled/sub_409910.c`

Факты:
- если `dword_4D7E70 == 0`, создаётся GPU-буфер через engine call.
- параметры буфера:
  - `dword_4D7E7C = 1000` (ёмкость в вершинах)
  - `dword_4D7E80 = 32` (stride bytes)
  - размер при создании: `dword_4D7E7C * dword_4D7E80` (bytes)
  - `dword_4D7E84 = 274 (0x112)` (FVF)
  - `dword_4D7E74 = 5` (primitive type, triangle strip)
- создание выполняется через `IDirect3DDevice9::CreateVertexBuffer(lengthBytes, usage, fvf, pool, &vb, 0)`:
  - `usage = 520 (0x208) = D3DUSAGE_DYNAMIC (0x200) | D3DUSAGE_WRITEONLY (0x8)`
  - `pool = 0 = D3DPOOL_DEFAULT`
- `flt_4D7E88` берётся как:
  - если `dword_4D7EE8` есть: `*(float *)(*(_DWORD *)(dword_4D7EE8 + 236) + 204)`
  - иначе `1.0`
- после чего вызывается `sub_4093B0()` для заполнения.

### 4.2 Рендер — `sub_409130`

Файл: `decompiled/sub_409130.c`

Факты:
- вызывает `sub_44F2A0(..., dword_4D7E70, dword_4D7E78, dword_4D7E74, dword_4D7E80, dword_4D7E84)`
- то есть буфер и счётчик вершин реально уходят в общий draw-процедурный путь `sub_44F2A0`.
