# F5: как записать «эталонные» числа из настоящей игры (IDA Pro)

## Зачем это вообще нужно

Вы хотите, чтобы ваша программа на TypeScript вела себя **точно так же**, как оригинальный `GoBonsai.exe`. Для этого нужны **образцы чисел** из настоящей игры в конкретные моменты — как эталон для сравнения.

**IDA Pro** здесь используется как **отладчик**: игра запускается, но в нужный момент **на паузу**. Пока пауза — можно посмотреть, какие числа лежат в памяти (время, энергия дерева и т.д.) и **записать их в блокнот**.

Это не «магия» и не требует быть программистом низкого уровня: нужно уметь нажимать несколько кнопок и переписывать цифры.

---

## Словарь (один раз)

| Слово | Простыми словами |
|--------|------------------|
| **Точка останова (breakpoint)** | Игра бежит, но как только доходит до выбранного места в коде — **стоп**, вы можете посмотреть память. В IDA точка останова — **красный кружок** на строке кода. Клавиша **F2** ставит/убирает точку. |
| **Функция `sub_40DC90`** | В декомпиле так называется кусок программы, который **каждый «тик» роста** обновляет мир и дерево. Мы останавливаемся на нём, чтобы в этот момент снять числа. |
| **Указатель** | В памяти хранится **адрес** — «куда смотреть дальше». Сначала читаем одно число (адрес мира), по нему — следующие числа (время, корень дерева). |

---

## Что записать в блокнот (результат работы)

Сделайте простую таблицу, например в Excel или текстовом файле:

| № | Что происходило в игре | Время (см. ниже) | Три числа с корня дерева |
|---|------------------------|------------------|---------------------------|
| 1 | Дерево уже на экране, первая остановка | … | 432: … 436: … 420: … |
| 2 | Подождали и снова поймали остановку | … | … |

**Время** для игры — это число по адресу «мир + 216» (ниже будет, как его получить). Оно **каждый раз при вызове `sub_40DC90` увеличивается на 1** — как счётчик дней в упрощённом виде.

**Корень дерева** — это объект в памяти; у него есть поля со смещениями **432, 436, 420** (энергия и т.д.) — их мы и переписываем.

---

## Перед началом

1. Используйте **один и тот же** файл `GoBonsai.exe` (не перепутайте с другой сборкой).
2. Лучше **не менять настройки игры** между запуском игры и записью чисел — иначе эталон и порт будут «про разное».
3. **Не ставьте** отладочные точки на **большие куски памяти целиком** — игра может зависнуть. Нам достаточно **одной точки на начале функции** `sub_40DC90` (это останов в **коде**, не в данных).

---

## Пошаговая инструкция

### Шаг 1. Открыть игру в IDA

1. Запустите **IDA Pro**.
2. Откройте файл **GoBonsai.exe**.
3. Дождитесь окончания анализа (полоска прогресса).

### Шаг 2. Найти функцию роста

1. Нажмите **G** (переход по адресу) или используйте поиск по имени.
2. Введите адрес **`0040DC90`** (так в проекте называется функция `sub_40DC90`; если IDA покажет другое имя — не страшно, главное **адрес**).
3. Вы попадёте в начало функции — это **дизассемблер**, список команд.

### Шаг 3. Включить отладку и запустить игру из IDA

1. Меню **Debugger → Select debugger** — выберите отладчик для Windows (обычно **Local Win32** для старого 32-бит exe).
2. **Debugger → Process options** — укажите путь к тому же **GoBonsai.exe**.
3. Нажмите **F9** (или меню **Debugger → Run**) — игра **запустится**.

Дождитесь, пока на экране будет **сцена с деревом** (как в обычной игре). Если останов произошёл слишком рано — нажимайте **F9** (продолжить), пока дерево не появится.

### Шаг 4. Поставить точку останова

1. Снова перейдите к адресу **`0040DC90`** (пока игра на паузе или в фоне — можно через список модулей вернуться в код).
2. Поставьте курсор на **самую первую строку** функции.
3. Нажмите **F2** — появится **красный кружок**.
4. Нажмите **F9**, чтобы игра снова пошла.

Теперь игра будет **снова и снова останавливаться** каждый раз, когда программа доходит до обновления роста. Это может быть **очень часто** — это нормально. Каждый раз, когда остановились, вы можете записать числа (следующий шаг) и снова нажать **F9**, чтобы продолжить.

Если остановок слишком много и вы не успеваете — просто **не спешите**: на одной остановке запишите всё нужное, потом **F9** один раз — и у вас уже «следующий тик».

### Шаг 5. Как достать числа (без формул в голове)

Когда выполнение **остановилось** (красная строка в IDA):

**А) Узнать адрес «мира» (число W)**

Смысл: по адресу **`004D7EE8`** в памяти лежит **не сам мир**, а **четыре байта**, которые вместе образуют **одно число** — адрес, по которому уже лежит объект «мир». Это число и есть **W**.

---

**Способ 1 — самый простой: окно Watch (рекомендуется)**

Не нужно вручную складывать байты.

1. Пока игра **на паузе** на точке останова, откройте окно **Watch** отладчика:  
   **Debugger → Debugger windows → Watch view** (название может быть чуть другим: «Watches»).

2. **Вариант А — одной строкой всё сразу (лучше всего).**  
   Добавьте в Watch **несколько строк** — по одному выражению в каждой (копируйте целиком):

   | Что нужно | Вставьте в Watch |
   |-----------|------------------|
   | Адрес мира **W** | `*(unsigned int*)0x004D7EE8` |
   | **Время** (пункт Б сразу) | `*(float*)(*(unsigned int*)0x004D7EE8 + 216)` |
   | Указатель **корня R** (пункт В) | `*(unsigned int*)(*(unsigned int*)0x004D7EE8 + 240)` |

   Если эти строки сработали — **пункты Б и В вы уже сделали**: время и **R** видны в таблице Watch. Переходите к **пункту Г** (энергия), см. ниже.

3. **Вариант Б — только W, дальше сами.**  
   Введите одну строку:

   `*(unsigned int*)0x004D7EE8`

   Нажмите Enter. В колонке значения появится **одно число** — это **W** (часто в **десятичном** виде). Запишите его и переходите к блоку **«Как из пункта А перейти к пункту Б»** ниже.

4. Если показывает **0** — нажмите **F9** и после следующей остановки снова посмотрите Watch.

Если выражения не принимаются — используйте **способ 2** или **скрипт из приложения** в конце файла.

---

**Способ 2 — вручную из Hex View (если Watch не сработал)**

1. **Debugger → Debugger windows → Hex view**.
2. Клавиша **G** (Jump) → введите **`004D7EE8`** → Enter.
3. Вы увидите **подряд четыре пары символов** — это четыре байта. Пример: `A8 7E 4D 00`.

   В Windows такие числа хранятся **в обратном порядке байт** (младший байт первым). Чтобы получить **W в шестнадцатеричном виде**, запишите байты **справа налево** как одно hex-число:

   - было: `A8 7E 4D 00`  
   - склеиваем: **`004D7EA8`** — это и есть адрес **W** в hex (пример, у вас будут другие цифры).

4. **Записать W можно так:** либо оставить как **`0x004D7EA8`** (подставьте свои цифры), либо перевести в десятичное число через калькулятор Windows: режим **Программист** → **HEX** → ввести ваше hex → переключить на **DEC** и записать.

5. Проверка: **W почти никогда не бывает нулём**, когда дерево уже на экране. Если получилось **00000000** — нажмите **F9** и прочитайте снова.

---

**Как из пункта А перейти к пункту Б (время)**

Теперь у вас есть **W** — одно целое число (адрес в памяти).

1. Нужен **новый адрес**: **W + 216** (именно **плюс двести шестнадцать**, не «по смещению в другой системе»).

2. **Удобнее всего — калькулятор Windows:**
   - Откройте **Калькулятор** → меню **≡** → **Программист**.
   - Если **W** у вас в **шестнадцатеричном** виде (например `004D7EA8`): включите **HEX**, введите это число, переключитесь на **DEC**, прибавьте **216**, запомните результат — снова переведите в **HEX** — это **адрес времени** для перехода в Hex View.
   - Если **W** в **десятичном** виде (из Watch): режим **DEC**, **W + 216**, результат переведите в **HEX** для поля перехода.

3. В **Hex View** снова **G** (Jump) → вставьте **получившийся адрес** (с префиксом **`0x`** или как принято в вашей IDA, часто достаточно `004D80D0` в hex).

4. На этом адресе лежат **следующие 4 байта** — это **время** в формате float. Его сложнее прочитать глазами; проще снова использовать **Watch**:

   `*(float*)(<адрес_времени>)`

   подставив вместо `<адрес_времени>` вычисленный **W+216** в виде, который принимает IDA (часто **`0x004D80D0`**).

**Б) Время (кратко)**

По адресу **W + 216** лежит **float** — **время симуляции**. Вы уже перешли на этот адрес шагом выше; запишите число через **Watch** `*(float*)0x........` или запустите **скрипт из приложения** — он выводит время сам.

**В) Корень дерева**

По адресу **W + 240** лежит ещё один указатель **R** (4 байта) — **корень дерева**. Запишите **R**.

Если **R = 0** — корень ещё не готов; снова **F9** и повторите на следующей остановке.

**Г) Три энергетических числа**

По смыслу это три float по адресам **корень + 432**, **+436**, **+420**.

Если **Watch** уже работал — добавьте три строки (копируйте целиком):

| Поле | Выражение для Watch |
|------|---------------------|
| энергия `+432` | `*(float*)(*(unsigned int*)(*(unsigned int*)0x004D7EE8 + 240) + 432)` |
| энергия `+436` | `*(float*)(*(unsigned int*)(*(unsigned int*)0x004D7EE8 + 240) + 436)` |
| энергия `+420` | `*(float*)(*(unsigned int*)(*(unsigned int*)0x004D7EE8 + 240) + 420)` |

Если **R = 0**, эти строки могут дать ошибку — сначала дождитесь ненулевого корня (**F9**).

Без Watch: вручную возьмите **R** из пункта В, в калькуляторе сложите **R + 432** (и +436, +420), переходите в Hex View по каждому адресу.

Заполните **одну строку** вашей таблицы.

### Шаг 6. Повторить для «чуть позже»

Нажмите **F9** много раз (или немного — как вам нужно). Снова дождитесь остановки, снова снимите **время** и **три числа**. Так у вас будет **второй снимок** — разница по времени покажет, сколько «тиков» прошло.

## Если Watch пишет «Cannot evaluate…»

Так бывает часто. Три типичные причины:

### 1. Игра не на паузе

Память процесса для Watch читается, когда отладчик **остановлен** (сработала точка останова, красная строка).

- Запустите из IDA (**F9**), дождитесь останова на `sub_40DC90`.
- Пока процесс просто «бежит» без останова — выражения в Watch часто дают **Cannot evaluate**.

### 2. ASLR — не тот адрес (самое частое)

В гайде адрес **`0x004D7EE8`** верен **как в файле**, если exe загружен с базы **`0x00400000`**.  
Windows часто грузит exe **с другой базы**. Тогда «старый» адрес **не отображается** в памяти процесса — Watch не может прочитать.

**Сделайте так:**

1. На паузе откройте **Debugger → Debugger windows → Modules** (список модулей).
2. Найдите **GoBonsai.exe** и колонку **Base** (например `0x00F00000`).
3. Посчитайте адрес глобала:

   **`адрес = Base + 0xD7EE8`**

   (формула: `0x004D7EE8 − 0x00400000 = 0xD7EE8` — это смещение внутри образа).

4. В **Hex View** (**G**) перейдите по **этому** адресу — должны быть 4 байта, не `??`.

Дальше читайте **W** из этих четырёх байт вручную или скриптом (приложение ниже с полем `BASE_EXE`).

### 3. Синтаксис Watch в IDA

Не все версии IDA понимают выражения вроде `*(unsigned int*)0x...`. Это **не ваша ошибка**. Используйте **Hex View** по адресу из пункта 2 или **скрипт Python** — они надёжнее.

## Если что-то пошло не так

| Проблема | Что сделать |
|----------|-------------|
| **Watch: Cannot evaluate** | Пауза на брейкпоинте + адрес **Base + 0xD7EE8** (см. блок выше). |
| **W или R равны нулю** | Игра ещё не дошла до сцены с деревом. Жмите **F9**, пока появится дерево. |
| **Адреса не сходятся** с книжкой | Убедитесь, что открыт **тот же exe**, что и в проекте декомпила. Редко помогает посмотреть **базу модуля** в **Debugger → Modules** — если игра грузится не с адреса `0x400000`, адреса в IDA могут сдвинуться; тогда ориентируйтесь на **имена** в IDA (**Names**) для `dword_4D7EE8`. |
| **Игра зависла** | Уберьте лишние точки останова на **память**; оставьте только **F2** на коде `sub_40DC90`. |
| **Остановов слишком много** | Это ожидаемо. Записывайте числа **на одной** остановке и реже жмите F9, или попросите знакомого сделать маленький скрипт из раздела ниже. |

## Скрипт (необязательно)

Если вы умеете запускать **IDAPython** (меню **File → Script file** или консоль Python в IDA), на паузе отладчика можно выполнить короткий скрипт — он напечатает время и одно число с корня. Код лежит в конце этого файла в блоке «Приложение» — его можно скопировать **целиком** и не разбираться, если не хочется.

---

## Что потом сделать с цифрами

Когда таблица заполнена:

1. Разработчик переносит числа в файл **тестов** в проекте (например константы в TypeScript).
2. В тесте программа на TS должна выдать **те же** числа при тех же условиях (тот же порядок вызовов и тот же генератор случайных чисел, см. `spec_ts_parity_verification.md`).

Так **F5** превращается из «идеи» в **конкретные проверки**: «наш порт совпал с оригиналом или нет».

## Приложение: короткий скрипт для IDA (Python)

Запускать **когда игра на паузе** на точке останова (после **F9**):

```python
import struct
import idc

def read_f32(addr):
    b = idc.get_bytes(addr, 4)
    return struct.unpack("<f", b)[0] if b else 0.0

# Если Watch не работает из-за ASLR: Modules → Base у GoBonsai.exe:
BASE_EXE = None   # например: 0x00F00000
RVA_GLOBAL = 0xD7EE8   # 0x004D7EE8 - 0x00400000

ea = idc.get_name_ea_simple("dword_4D7EE8")
if ea == idc.BADADDR:
    ea = 0x004D7EE8
if BASE_EXE is not None:
    ea = BASE_EXE + RVA_GLOBAL

W = idc.get_wide_dword(ea)
print("Читаем с", hex(ea), "-> W =", hex(W) if W else W)
if W:
    t = read_f32(W + 216)
    R = idc.get_wide_dword(W + 240)
    print("Время (W+216):", t, "  Корень R:", hex(R) if R else R)
    if R:
        e420 = read_f32(R + 420)
        e432 = read_f32(R + 432)
        e436 = read_f32(R + 436)
        print("root+420:", e420)
        print("root+432:", e432)
        print("root+436:", e436)
    else:
        print("R=0 — корень ещё не готов, нажмите F9 и повторите.")
else:
    print("W=0: выставьте BASE_EXE из Modules или нажмите F9 когда дерево на экране.")
```

Если **`BASE_EXE`** не задан и **`0x004D7EE8`** не подходит — задайте базу вручную из **Modules**.
