# Верификация паритета TS ↔ бинарник

## 1. Генератор случайных чисел

В оригинале используется **`rand()` из MSVCRT** (`rand()/32767.0` в ветвлении, листьях и др.).

В проекте: `gobonsai-web/src/engine/MSVCRand.ts` — линейный конгруэнтный генератор с теми же константами, что и MSVC (`214013`, `2531011`, выход `(state >> 16) & 0x7FFF`).

**Требование паритета:** один экземпляр (или явная цепочка `srand`/`rand`) на ту же **последовательность вызовов**, что и в C при том же сценарии (см. §3).

## 2. Порядок кадра (минимальный чеклист)

Проверять в таком порядке (от общего к частному):

1. Обновление мира / время `dword_4D7EE8+216` и производные годы.
2. `sub_40DC90` (или эквивалент): `root+432`, virtual `+36`, баланс `+420/+436`, `energyLeak`, сезон `sub_40EE60`.
3. Матрицы: `+32` локальная геометрия, `+16` world matrix по иерархии.
4. Листья: очередь `sub_40E0A0` / трассировка `sub_40E460` (см. спеки листа).
5. Геометрия `sub_4093B0`, submit `sub_44F2A0`.

Любое несоответствие порядка даёт «логика как в C, дерево не растёт / ломается» без ошибки в формулах.

## 3. Сценарии регрессии (численные якоря)

Идея: зафиксировать **seed**, **INI по умолчанию**, **время** и снять снимки:

| Сценарий | Что сравнить | Источник истины |
|----------|----------------|-----------------|
| Старт из семени | `+432`, число секций, первый bud после порога | `spec_initial_sprout_growth.md` |
| Один год Twig | вызов `sub_417F40` не чаще ожидаемого; `+440` обновлён | `spec_branching_dispatcher_sub_417F40.md` |
| Лист | production `+420`, интеграл `+424`, кватернион после N кадров при фиксированном свете | `spec_leaf_update_sub_416510.md` |
| Энергия | после кадра `+436` и утечка | `spec_update_growth_sub_40DC90.md` |

Конкретные числа записывать в тест по мере появления **записи с эталонного запуска** `.exe` (лог памяти / отладчик), а не «на глаз».

## 3.1 Зафиксированный эталон (из IDA)

Первые снимки с оригинала лежат в коде: `gobonsai-web/src/engine/__fixtures__/parityReferenceFromExe.ts` — два состояния при `worldTime` **108** и **115** (разница **7** тиков `sub_40DC90`), поля `root+420/+432/+436`.

**Автотесты в репозитории:** `gobonsai-web` — `npm test`.

- **`MSVCRand.test.ts`** — последовательность MSVC при `seed=1` (CI всегда зелёный).
- **`parityVerification.test.ts`** — контракт фикстуры (дельта A/B = 7) + smoke harness (`simulationDay`); сравнение с `paritySnapshotA/B` включено только при **`GOBONSAI_EXE_PARITY=1`** (ожидаемо падает, пока движок не совпадёт с exe).

## 4. Автоматизация

- **CI:** `npm test` — RNG + лёгкие проверки без эталона exe.
- **Паритет с exe:** `GOBONSAI_EXE_PARITY=1 npm test` — жёсткое сравнение `root+420/+432/+436` с `parityReferenceFromExe.ts` после 108 и 115 кадров (минимальный harness без сохранений/UI).

## Статус

- Контракт RNG: **задокументирован**, **покрыт тестом** в CI.
- Числовые эталоны: **заполняются по мере снятия трасс с оригинала**; регрессия — опционально через `GOBONSAI_EXE_PARITY`.
