# Ветвление: диспетчер `sub_417F40` и три реализации

Цель паритета: один раз в год (в смысле «годовых тиков» относительно `this+440`) на **Twig** (`sub_417C90`) вызывается `sub_417F40(this)`, который выбирает **апикальное** (`sub_417FF0`), **боковое** (`sub_4188E0`) или **упрощённое без апикала** (`sub_418660`).

## Вызов

- `sub_417C90` (update Twig) → `sub_417F40(this)` — см. `decompiled/sub_417C90.c`.

## `sub_417F40` — логика выбора

Файл: `decompiled/sub_417F40.c`.

Обозначения:

- `worldTime = *(dword_4D7EE8 + 216)` — время мира.
- `yearsSinceBranch = (int)(worldTime/365) - (int)((worldTime - *(this+440)) / 365)` — целое число «полных лет» с момента, записанного в `this+440` (время последнего ветвления на этом участке).

Псевдокод:

```
if yearsSinceBranch < 1: return

if yearsSinceBranch > 3 && *(this+444) > 0.2:
    sub_4188E0(this)   // боковая ветка
else:
    childCount = *(this+172)
    if childCount <= 1 && (childCount != 1 || (sub_416300(firstChild) && sub_415470(sub_416300(...)))):
        sub_417FF0(this)
```

### Вспомогательные сабы

- **`sub_415470(section)`** — возвращает то же «число лет с `+440`», что считает диспетчер, но для **другой** секции (аргумент — указатель на дочерний узел из `sub_416300`). Файл: `decompiled/sub_415470.c`.
- **`sub_416300(section)`** — если секция ненулевая и `byte_4D822A[11 * typeId] != 0`, возвращает указатель; иначе `0`. То есть фильтр по типу секции (`decompiled/sub_416300.c`).

Условие ветки `sub_417FF0` намеренно громоздкое: при `childCount == 1` требуется, чтобы первый ребёнок прошёл проверку типа **и** имел «год от ветвления» в смысле `sub_415470` (согласование с соседним побегом).

## `sub_4188E0` — боковая ветка

Файл: `decompiled/sub_4188E0.c`.

- Триггер по энергии: `*(this+432) - *(this+436) >= minBudEnergy` (`flt_4D862C`).
- Вероятность шага пропорциональна излишку энергии, с порогом `flt_4D8630`, клип `v14` в `[0, 0.003]`, сравнение с `rand()/32767`.
- Создаётся новая секция через `sub_4158D0` / `sub_4159C0`, списывается энергия в `this+436`, выставляются `child+424/+428` с нормировкой по сумме `+428` сиблингов (цикл по списку `*(this+168)`).

Используется **`rand()`** (не inline другого PRNG) — для TS нужен `MSVCRand` в том же порядке вызовов.

## `sub_417FF0` — апикальное / парное ветвление при наличии «оси»

Файл: `decompiled/sub_417FF0.c`.

- Расход энергии: `v17 = apicalGrowthStrength * (availableEnergy) * v33`, где `v33` — степень от `(this+484 - 1)` и `flt_4D8638`.
- Если `*(this+172) != 0` (есть дети): сложная геометрия (инверсия `this+104`, углы, `randomBudRotation`), создаются **два** новых побега с матрицами.
- Если `*(this+172) == 0`: **`sub_418660(this)`** — ветка «без апикальной оси».

## `sub_418660` — упрощённое ветвление (листовая/пустая ось)

Файл: `decompiled/sub_418660.c`.

- Случайный yaw, радиус в плоскости, один дочерний сегмент; при достаточной энергии и `rand() < 0.2` — второй сегмент с противоположным направлением.

## Связь с аккумуляторами

Потребление `this+436` и присвоение `child+424/+428` согласованы с `GUIDES/spec_energy_accumulators_420_436.md`.

## Статус

- Диспетчер и три ветки: **замкнуты** по декомпилу; для TS остаётся строго соблюдать порядок `rand()` и поля `this+440`, `this+444`, `this+172`, `this+484`.
