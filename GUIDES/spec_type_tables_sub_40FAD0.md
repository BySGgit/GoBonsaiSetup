## 1. Структура таблиц

Три параллельных массива с шагом **11 байт** на тип секции:

| Базовый адрес | Смещение от базы | Семантика |
|---------------|------------------|-----------|
| `byte_4D8220` | `[11 * typeId + 0]` | Flag #0: «тип зарегистрирован» |
| `byte_4D8225` | `[11 * typeId + 0]` = `byte_4D8220[11*typeId + 5]` | Flag #5: «копировать свет/level от родителя» в `sub_413F50` |
| `byte_4D8227` | `[11 * typeId + 0]` = `byte_4D8220[11*typeId + 7]` | Flag #7: «свет затухает через lightDecayAmount» в `sub_40E460` |
| `byte_4D8229` | `[11 * typeId + 0]` = `byte_4D8220[11*typeId + 9]` | Flag #9: ветка `sub_414CE0` (копия +428 от родителя) |
| `byte_4D822A` | `[11 * typeId + 0]` = `byte_4D8220[11*typeId + 10]` | Flag #10: ветка `sub_414CE0` (pow(+424)) и `sub_40E460` (геом. экранирование) |

**Важно**: `byte_4D8225`, `byte_4D8227`, `byte_4D8229`, `byte_4D822A` — это **не отдельные массивы**, а срезы внутри одной 11-байтной таблицы по типу (страйд 11).
