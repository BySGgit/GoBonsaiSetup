# Спека: кривая материала `sub_4151B0` + палитра `sub_413DF0`

Файлы: `decompiled/sub_4151B0.c` (95 строк), `decompiled/sub_413DF0.c`.

Цель: полный разбор формулы `(1 - (1 - x)^6)` и палитры цветов для достижения паритета цвета.

---

## 1. Сигнатура

```
RGBA* sub_4151B0(float* section, RGBA* out)
```

- `section` — указатель на объект секции (как `float*`, смещения в `float`-единицах)
- `out` — выходной RGBA (4 × float)
- Читает глобальный `dword_4D8644` — режим окраски

---

## 2. Режим окраски `dword_4D8644`

Переключается горячими клавишами (case 21–26 в WndProc) или через UI:

| Значение | Описание | Поле секции (`float*` индекс) | Byte offset |
|----------|----------|-------------------------------|-------------|
| 0 | (default — ранний выход) | — | — |
| 1 | (default — ранний выход) | — | — |
| 2 | Energy production | `[105]` | +420 (`energyProduction420`) |
| 3 | Energy budget | `[108]` | +432 (`energyBudget432`) |
| 4 | Energy spent | `[109]` | +436 (`energySpent436`) |
| 5 | Health energy (прямой) | `[114]` | +456 (`healthEnergy456`) |

> Режимы 0 и 1 — `return out` без изменений (оригинальный цвет).

---

## 3. Формула кривой

### Режимы 2, 3, 4 (с кривой `(1 - (1-x)^6)`):

```
x = section[fieldIndex] / 20.0
x = clamp(x, 0.0, 1.0)
t = 1.0 - pow(1.0 - x, 6.0)
t = clamp(t, 0.0, 1.0)
color = sub_413DF0(t)
```

### Режим 5 (health — без кривой):

```
t = section[114]                               // healthEnergy456 — уже в [0, 1]
t = clamp(t, 0.0, 1.0)
color = sub_413DF0(t)
```

### Визуальное описание кривой

`f(x) = 1 - (1 - x)^6` — это **ease-out** кривая степени 6:
- При x=0 → f=0
- При x=0.1 → f ≈ 0.47
- При x=0.2 → f ≈ 0.74
- При x=0.5 → f ≈ 0.98
- При x=1.0 → f=1.0

Кривая быстро выходит на плато, т.е. даже небольшая энергия (`x > 0.1`) уже даёт почти «здоровый» цвет. Это создаёт визуально контрастное разделение: «мёртвый» ← 0 → «живой».

## 4. `sub_413DF0` — палитра цветов

```
float* sub_413DF0(float* out, float t):
    // Lazy init — 8 ключевых точек палитры (32 float = 8 × RGBA):
    static palette[0..31]:
        [0]  = {R=0.0,  G=0.0,  B=0.0,  A=1.0}    // t=0.0: чёрный
        [4]  = {R=1.0,  G=1.0,  B=0.0,  A=1.0}     // ...жёлтый
        [8]  = {R=0.0,  G=0.0,  B=1.0,  A=1.0}     // ...синий??? (flt_4D8D38=0)
        // Подробнее ниже

    sub_40FDE0(8, out, &palette[0], t)               // интерполяция 8-точечной палитры
    return out
```

### Полная палитра (из lazy init `sub_413DF0`)

| Индекс | Offset | R | G | B | A | Визуальный цвет |
|--------|--------|---|---|---|---|-----------------|
| 0 | `flt_4D8D20` | 0.0 | 0.0 | 0.0 | **1.0** | Чёрный (мёртвый) |
| 1 | `flt_4D8D30` | **1.0** | 0.0 | 0.0 | **1.0** | Красный |
| 2 | `flt_4D8D40` | **flt_4D63B0** | 0.0 | 0.0 | **1.0** | Тёмно-красный (flt_4D63B0 ≈ 0.5) |
| 3 | `flt_4D8D50` | **1.0** | **1.0** | **1.0** | **flt_4D62E8** | Белый с alpha из INI |
| 4 | `flt_4D8D60` | 0.0 | **0.2** | **1.0** | 0.0 | Сине-зелёный |
| 5 | `flt_4D8D70` | **1.0** | **1.0** | 0.0 | 0.0 | Жёлтый |
| 6 | `flt_4D8D80` | 0.0 | **0.7** | 0.0 | 0.0 | Зелёный |
| 7 | `flt_4D8D90` | **1.0** | **1.0** | **1.0** | **1.0** | Белый |

> **Примечание**: порядок float в RGBA: `[offset+0]=R, [offset+4]=G, [offset+8]=B, [offset+12]=A`. Реконструкция из `flt_4D8D20..4D8D9C`:
> - `flt_4D8D20=0, flt_4D8D24=0, flt_4D8D28=0, flt_4D8D2C=1` → key 0
> - `flt_4D8D30=1, flt_4D8D34=0, flt_4D8D38=0, flt_4D8D3C=1` → key 1
> - `flt_4D8D40=flt_4D63B0, flt_4D8D44=flt_4D63B0, flt_4D8D48=0, flt_4D8D4C=1` → key 2 (поправка: G тоже = flt_4D63B0)

**Уточнение по key 2**: из кода `flt_4D8D44 = flt_4D63B0` — IDA показывает `flt_4D8D44 = flt_4D63B0`, значит `G = flt_4D63B0` тоже. Это даёт не тёмно-красный, а тёмно-жёлтый / коричневый (если flt_4D63B0 ≈ 0.5 → `(0.5, 0.5, 0, 1)` = оливковый).

### `sub_40FDE0(8, out, palette, t)`

Стандартная интерполяция по 8-точечной палитре (документирована в `spec_geometry_sub_4093B0.md`). Параметр `t ∈ [0, 1]` линейно отображается на сегмент палитры, и между двумя ключевыми точками выполняется `lerp` RGBA.

---

## 5. Режимы 0 и 1

При `dword_4D8644` ∉ {2, 3, 4, 5} функция возвращает `out` **без изменений** (ранний `return a2`). Это означает, что в режимах 0/1 используется цвет, уже установленный вызывающей draw-процедурой (или предыдущим вызовом).

**Уточнение по mode 1**: в `full.c` ~10901 есть дополнительная проверка `if (dword_4D8644 != 1)` перед вызовом `sub_4151B0`, что подтверждает: при mode=1 `sub_4151B0` вообще не вызывается.

## 6. Связь с TS

| C | TS | Статус |

### TS рекомендация

```typescript
const MATERIAL_PALETTE = [
  { r: 0.0,       g: 0.0,       b: 0.0, a: 1.0 },   // 0: чёрный (мёртвый)
  { r: 1.0,       g: 0.0,       b: 0.0, a: 1.0 },   // 1: красный
  { r: GC.BASE_GROWTH, g: GC.BASE_GROWTH, b: 0.0, a: 1.0 }, // 2: оливковый
  { r: 1.0,       g: 1.0,       b: 1.0, a: GC.FLT_4D62E8 }, // 3: белый (alpha из INI)
  { r: 0.0,       g: 0.2,       b: 1.0, a: 0.0 },   // 4: сине-зелёный
  { r: 1.0,       g: 1.0,       b: 0.0, a: 0.0 },   // 5: жёлтый
  { r: 0.0,       g: 0.7,       b: 0.0, a: 0.0 },   // 6: зелёный
  { r: 1.0,       g: 1.0,       b: 1.0, a: 1.0 },   // 7: белый
];

function materialCurve(value: number): number {
  const x = Math.max(0, Math.min(1, value / 20.0));
  return 1.0 - Math.pow(1.0 - x, 6.0);
}

function resolveMaterialColor(
  section: TreeSection,
  mode: number,
  out: { r: number; g: number; b: number; a: number }
): boolean {
  let t: number;
  switch (mode) {
    case 2: t = materialCurve(section.energyProduction420); break;
    case 3: t = materialCurve(section.energyBudget432); break;
    case 4: t = materialCurve(section.energySpent436); break;
    case 5:
      t = Math.max(0, Math.min(1, section.healthEnergy456));
      break;
    default:
      return false;
  }
  sub40FDE0Interpolate(8, out, MATERIAL_PALETTE, t);
  return true;
}
```
