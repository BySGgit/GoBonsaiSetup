# Спека: виртуальный слот vtable `+44` (что вызывается как `(*this + 44)(this)`)

Цель: зафиксировать “истину” о загадочном вызове `(*(fn**)(*(_DWORD*)this + 44))(this)` в декомпиле.

Важно:
- `+44` — это **смещение в vtable**, не в объекте.
- индекс = `44 / 4 = 11` → **12-й указатель** таблицы виртуальных функций.

---

## 1) Что стоит на `+44` по типам `TreeSection*` (из `decompiled/rdata.txt`)

### `TreeSection` (base vtable `.rdata:004B7EF4`)

- vtable[11] (`+44`) указывает на **`Concurrency::...::decline_incoming_messages(void)`** (адрес `.text:00414260`).
- В `decompiled/.text.txt` это помечено как **COLLAPSED FUNCTION [00000008 BYTES]**.
- Реальное тело (из IDA) для `.text:00414260`:
  - `mov byte ptr [ecx+0ECh], 1`

Сопоставление с `TreeSection`:
- `0xEC = 236` байт, а в `sub_413F50` есть явное поле `*(BYTE *)(a1 + 236) = 0`.
- Значит, `(*this + 44)(this)` для base/leaf/twig/stump **ставит флаг `this+236 = 1`**.

Практический смысл (по потребителям флага):
- `sub_414270` и `sub_418E40` обходят детей и затем делают `if (*(_BYTE *)(this + 236)) sub_40EEE0(this);`.
- То есть вызов `(*this+44)` для leaf/twig/stump/base — это **отложенное “пометить к обработке”**, а реальная обработка делается позднее через `sub_40EEE0`.

### `TreeSectionSeed` (`.rdata:004B8054`)

- vtable[11] (`+44`) → `nullsub_1` (`decompiled/nullsub_1.c`, пустое тело).

Интерпретация: для Seed этот слот **ничего не делает** (скорее всего “нельзя удалить корень таким способом” или “no-op”).

### `TreeSectionBud` (`.rdata:004B7F3C`)

- vtable[11] (`+44`) → `sub_415ED0` (`decompiled/sub_415ED0.c`).
- `sub_415ED0`:
  - если есть родитель `this+180`, вызывает `sub_450C80(this, parent)` (detach из списка),
  - затем вызывает виртуальный `(*this + 8)` (обычно деструктор/удаление).

Интерпретация: для Bud слот `+44` — это **“удалить/отсоединить и уничтожить”**.

### `TreeSectionLeaf` (`.rdata:004B800C`)

- vtable[11] (`+44`) → тот же **`Concurrency::...decline_incoming_messages`** (`.text:00414260`, collapsed 8 bytes).

Интерпретация:
- В `sub_416510` ранний выход делает `(*this+44)(this); return;`.
- Для Leaf этот вызов **не удаляет** секцию: он **ставит флаг `this+236 = 1`** (см. `.text:00414260`).
- Реальное удаление для Leaf, вероятно, делается через другой слот (например, `+8`/`+44` у других типов) или через родителя/обрезку.

### `TreeSectionTwig` (`.rdata:004B8134`) и `TreeSectionStump` (`.rdata:004B809C`)

- vtable[11] (`+44`) у обоих также указывает на **`Concurrency::...decline_incoming_messages`** (`.text:00414260`, collapsed 8 bytes).

---

## 2) Что это значит для трактовки декомпила

- Нельзя автоматически считать, что `(*this+44)` = “удалить объект”.
- Для Bud это действительно **kill/detach** (`sub_415ED0`).
- Для Leaf/Twig/Base это **установка флага `this+236 = 1`**, а не удаление.
