# Журнал истины из IDA Pro (MCP) → порт TS

**Назначение:** фиксировать выводы сессий с IDA (декомпил, xref’ы, callees), чтобы не терять контекст между чатами и не дублировать вызовы MCP.  
**Правило:** новая запись добавляется **в конец** раздела «Записи»; краткая отсылка в коде TS — `// IDA truth: GUIDES/ida_extracted_truth_log.md § <якорь>`.

**Связанные файлы:** `sub_to_ts_coverage_map.md`, `spec_ts_parity_verification.md`, `spec_f5_ida_capture_workflow.md`, скилл `.cursor/skills/ida-function-analysis/SKILL.md`.

---

## Как добавлять запись

1. Якорь: `### sub_XXXXXX — <краткое имя> (<YYYY-MM-DD>)`
2. Обязательные поля: **Адрес**, **Источник** (MCP `decompile` / ручной IDA), **Контракт** (входы/выходы/побочные эффекты), **Callees**, **Xrefs to**, **Соответствие TS**, **Открыто / следующий шаг в IDA**
3. По возможности — цитата псевдокода **только обрывками** (полный текст тянуть из `.i64` / `decompiled/*.c`, не раздувать репозиторий).
