# GoBonsai RE (Educational Reverse Engineering Project)

Веб-версия классической игры GoBonsai, воссозданная на основе анализа оригинального бинарного кода (C/DirectX) и портированная на современный стек (Vue 3, Three.js, TypeScript).

## Основные особенности

- **Процедурная генерация**: Дерево растет уникальным образом каждый раз, используя алгоритмы ветвления, извлеченные из `sub_4093B0`.
- **Биологическая симуляция**: Система метаболизма (энергия, здоровье, фотосинтез) на основе `sub_416510` и `sub_4417C0`.
- **Визуальные эффекты**:
  - Динамическое старение коры (появление трещин и мха со временем).
  - Сезонные изменения цвета листьев (осень/зима).
  - Реалистичное опадание листьев при низком здоровье.
  - Процедурные корни и рельефная почва с камнями.
- **Атмосферный звук**: Процедурный шум ветра и пение птиц, зависящее от времени суток.
- **Интерактивность**: Полив дерева и управление освещением в реальном времени.
- **Сохранение прогресса**: Состояние дерева автоматически сохраняется в `localStorage`.

## Технический стек

- **Frontend**: Vue 3 (Composition API), Vite, TailwindCSS.
- **3D Engine**: Three.js.
- **Icons**: Lucide Vue Next.
- **Language**: TypeScript.

## Технические детали (Reverse Engineering)

В ходе работы над проектом были воссозданы следующие внутренние механизмы оригинальной игры:

### 1. Система роста и сглаживания (sub_408D60)
Оригинальный код использует систему целевых параметров (`target age`, `target thickness` и т.д.). В каждом кадре текущие значения стремятся к целевым с фиксированным шагом. Мы перенесли это в `BonsaiController.updateAnimation`, адаптировав под `deltaTime` для плавности на мониторах с высокой частотой обновления.

### 2. Структура дерева (sub_4093B0)
Ветвление реализовано как рекурсивная структура `TreeSection`. Каждая ветка имеет свой уровень (`level`), от которого зависят коэффициенты масштабирования и вероятность появления новых побегов или листьев. Мы также воссоздали "узловатость" веток, разделяя каждый сегмент на 3 части со случайными микро-отклонениями.

### 3. Метаболизм (sub_416510 / sub_417C90)
- **Энергия**: Накапливается при достаточном освещении (фотосинтез) и тратится на поддержание жизни и рост.
- **Здоровье**: Снижается, если энергия падает ниже критического порога.
- **Восстановление**: Полив (`interact`) напрямую восстанавливает энергию и здоровье.

### 4. Процедурная генерация
- **MSVCRand**: Используется алгоритм LCG из библиотеки времени выполнения Microsoft Visual C++ для идентичности случайных чисел оригиналу.
- **Шейдеры**: Кора и листья генерируются процедурно. Шейдер коры имитирует трещины и мох, зависящие от возраста дерева.

## Как запустить

### Предварительные требования
- Установленный [Node.js](https://nodejs.org/) (рекомендуется LTS версия).

### Установка и запуск
1. Перейдите в папку проекта:
   ```bash
   cd gobonsai-web
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите сервер для разработки:
   ```bash
   npm run dev
   ```
4. Откройте браузер по адресу `http://localhost:5173`.

## Управление
- **Клик по экрану**: Полить дерево (восстанавливает энергию и здоровье).
- **Слайдер Time Speed**: Ускорение течения времени.
- **Manual Light**: Переключение на ручное управление освещением для контроля фотосинтеза.
